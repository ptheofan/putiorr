Core model

putiorr pretends to be Transmission for the *arr side, sends magnet and torrent grabs to put.io, then downloads completed put.io files into a local download target. Completed-download handling in Radarr, Sonarr, Lidarr, and similar apps then imports from that same mounted path into the normal media libraries. 

put.io -> putiorr -> shared download target -> *arr import -> media library
